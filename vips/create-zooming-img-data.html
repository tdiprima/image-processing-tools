jpstroop commented on Dec 11, 2013

Loris, which implements the IIIF image API uses the JPEG2000 (aka 'JP2') image format, which (depending on how you compressed your files, but usually) has tiles embedded in the image. We store one JP2, and the tiles are extracted on the fly as needed. We generally cache the tiles after they've been extracted once, and we do get a performance boost from it, but this is really only necessary for medium to large (say > 3200px) images.

The IIIF image metadata looks like this (the server also extracts and caches this):

http://libimages.princeton.edu/loris/new_jp2s%2Fpudl0052%2F6131707%2F00000001.jp2/info.json

and here's a demo, fwiw:

http://libimages.princeton.edu/osd-demo/?feedme=new_jp2s%2Fpudl0052%2F6131707%2F00000001.jp2

We use the Kakadu JP2 binaries to compress (from TIFF) and expand (generally to JPEG) our image files.
